Dataverse Schema

All agents read from and write to Dataverse. Data flows between agents via Dataverse records, not point-to-point. All data is synthetic for the hackathon.

TableKey FieldsWritten ByRead By
Patients ID, name, DOB, conditions, medications, allergies Pre-loaded (synthetic) Triage, Documentation, Referral

Referrals ID, patient, clinician, specialty, letter, status (draft/sent), dates Referral Agent —
Audit Logs ID, agent, action, entity ref, user, timestamp All agents Admin (reporting)

Key Relationships

  • Triage → Encounter: Triage record pre-fills SOAP (S); linked via patient + triage ref
  • Triage → Appointment: Triage outcome triggers booking; linked via triage ref
  • Encounter → Referral: Referral flag in encounter triggers Referral Agent; linked via encounter ID
  • All → Audit Logs: Every agent action creates an audit record

Security & Governance

Authentication

Microsoft Entra ID for clinicians (Teams) and patients (chatbot)

Authorisation

Dataverse RBAC — clinicians see patient data; patients see own data only

Data Protection

Encryption at rest + transit; synthetic data only for hackathon

Audit Trail

Immutable log of all agent actions and clinician approvals

Responsible AI

Agents handle admin only — no clinical decisions; nothing sent without approval

Compliance Ready

Designed for NHS/HIPAA-level controls in production
