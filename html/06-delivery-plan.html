Delivery Plan

AgentCare — Hackathon MVP scope, effort estimates, and sprint plan

1. Hackathon Scope

Focus is administrative automation — no clinical decision-making. Three sub-processes, three agents, covering triage, scheduling, and documentation.

Sub-ProcessAgentScopeRationale
Patient Intake & Triage Triage Agent In scope Emergency scan (before sign-in), symptom collection, history-aware urgency, care advice, receptionist escalation, FAQ
Appointment Management Scheduling Agent In scope Self-service booking, confirmations, waitlist management
Consultation & Documentation Documentation Agent In scope Pre-consult briefing, SOAP drafting from transcript, entity extraction, patient email

3. MVP Definition

In Scope (Must Have)

  • Triage Agent — Emergency scan (pre-auth), symptom Q&A, history-aware urgency, care advice, receptionist escalation
  • Scheduling Agent — Appointment booking, confirmations, waitlist
  • Documentation Agent — Pre-consult briefing, SOAP drafting from transcript, entity extraction, patient email
  • Dataverse — 5 tables (Patients, Triage Records, Appointments, Encounters, Audit Logs)
  • Power Automate — Agent flows for emails, confirmations, notifications
  • Audit trail — All agent actions logged

Out of Scope (Post-Hackathon)

  • Clinical decision support (drug interactions, guideline matching)
  • Referral management (post-hackathon scope)
  • Real EMR/PAS integration (use synthetic data)
  • eRS (e-Referral Service) live integration
  • Patient-facing mobile app (use web chatbot)
  • Multi-practice deployment
  • Power BI dashboards

Key Assumptions

  • Power Platform dev environment available from day 1
  • Copilot Studio licences provisioned
  • Synthetic clinical data only (no real PII/PHI)
  • No external system integration — Dataverse-only
  • Chatbot deployed on practice website (Copilot Studio web channel)
  • Clinician uses Teams (Copilot chat channel)
  • Outlook for all email notifications
